# 01. Product Requirements

## 사용자

주요 사용자는 첫 월급을 받았거나 곧 받을 사회초년생 1~3년차다. 금융 정보 탐색 경험은 있지만, 자신의 상황에서 무엇부터 해야 하는지 결정하기 어렵고 실행을 미루는 경향이 있다.

## 사용자 문제

- 지금 내 상황에서 가장 먼저 해야 할 금융 행동을 모른다.
- 금융 용어와 제도가 부담스럽다.
- 콘텐츠를 봐도 가입, 신청, 설정 같은 실제 행동으로 이어지지 않는다.
- 한 번 시작해도 꾸준히 지속할 동기가 약하다.

## MVP 기능

### 1. 간편 입력 온보딩

- 자산, 월소득, 고정지출, 저축 여부, 청약/연금/투자 경험, 목표를 입력한다.
- 초기 MVP는 계좌 연동 없이 자가 입력을 사용한다.
- 자연어 입력과 선택형 입력을 함께 지원할 수 있다.

### 2. 5단계 자산 형성 진단

- 비상금, 청약·연금, 적금·예금, 투자 기초, 자산 분산 순서로 충족 여부를 점검한다.
- 사용자의 현재 상태를 "집중 단계"로 표현한다.
- 진단 결과에는 이유, 다음 행동, 주의사항을 포함한다.
- 진단 결과에는 어떤 입력값과 룰이 사용되었는지 사용자가 이해할 수 있는 수준으로 설명한다.
- 원본 금융정보를 과도하게 저장하지 않고 `비상금 부족`, `청약 미점검`, `저축률 미설정` 같은 상태값을 우선 사용한다.

### 3. 주간 챌린지 추천

- 사용자의 집중 단계, 목표, 진행 이력에 맞춰 주 1개 챌린지를 추천한다.
- 챌린지에는 예상 소요시간, 난이도, 준비물, 실행 링크, 완료 기준이 포함된다.
- 첫 챌린지는 완료 장벽이 낮은 행동을 우선한다.

### 4. 챌린지 진행

- Engage: 이번 주 해야 할 행동과 이유 제시
- Investigate: 짧은 학습 콘텐츠와 공식 출처 제공
- Act: 실제 실행 방법과 외부 링크 제공
- Reflect: 완료 후 회고 질문과 다음 액션 제안
- 완료 방식은 `계획 완료`, `실행 완료`, `보류 사유 기록` 중 하나로 남긴다.
- 챌린지마다 사용자가 실제 행동을 했는지 확인할 수 있는 자기 체크 질문을 둔다.

### 5. 효과 계산과 성장 기록

- 챌린지 완료 후 사용자가 얻은 결과를 숫자로 보여준다.
- 예: 월 절감액, 연간 절감액, 월 저축 목표, 비상금 달성 예상 기간
- 완료 기록, 연속 수행 일수, 누적 행동 수를 간단히 보여준다.
- 스트릭은 실패를 벌주는 방식이 아니라 재시작을 돕는 방식으로 설계한다.

### 6. AI 코칭

- 사용자의 질문에 공식 자료 기반으로 답한다.
- 사용자의 맥락에 맞춰 용어를 쉽게 설명한다.
- 완료 회고를 통해 다음 행동으로 이어지게 한다.
- 챌린지 완료 후 사용자가 한 행동을 구체적으로 칭찬하고 다음 행동을 하나만 제안한다.
- 같은 자산 단계 안에서도 사용자의 보류 사유, 이해도, 완료 이력에 따라 다른 안내를 제공한다.

## 후순위 기능

- 계좌 연동 기반 자동 진단
- 금융상품 제휴 연결
- 또래 비교 통계
- ETF 구성 종목/수수료 검색
- 월간 자산 성장 리포트
- 유료 구독용 심층 AI 코칭
- 자산 시뮬레이션
- 행동 변화 RCT 설계
- SNS 공유 자동화
- 친구 초대 리워드
- AI 캐릭터 영상 생성

## 비기능 요구사항

- 금융 정보는 출처와 기준일을 함께 제공한다.
- 사용자의 민감정보 수집을 최소화한다.
- AI 응답은 금융상품 추천, 투자자문, 가입 대행으로 오해되지 않게 설계한다.
- 서비스는 모바일 우선으로 설계한다.

## MVP 챌린지 후보

초기에는 복잡한 상품 가입보다 짧고 안전하게 완료할 수 있는 행동을 우선한다.

- 비상금 목표 만들기
- 월급날 자동이체 계획 만들기
- 고정지출 한 개 줄이기
- 청약통장 상태 점검하기
- 현실적인 월 저축률 정하기
- 청년 자산 형성 정책 자격 간이 확인하기
