# 02. Challenge System

## 설계 원칙

챌린지는 공부 과제가 아니라 실제 금융 행동이다. 사용자는 챌린지를 수행하면서 필요한 만큼만 배우고, 실행 결과를 통해 자신의 금융 역량이 쌓이는 경험을 해야 한다.

모든 MVP 챌린지는 아래 조건을 만족해야 한다.

- 사용자가 5~15분 안에 첫 결과를 만들 수 있다.
- 완료 기준이 화면에 명확히 보인다.
- 완료 후 금액, 상태, 다음 행동 중 최소 하나가 남는다.
- 민감한 증빙 제출 없이도 진행할 수 있다.
- 사용자가 보류하더라도 보류 사유를 기록하고 다음 행동으로 이어질 수 있다.

## CBL 흐름

### Engage

사용자의 현재 자산 단계에서 이번 주에 가장 중요한 행동을 하나 제안한다.

예: "이번 주 미션: 청약통장 가입 가능 여부 확인하기. 예상 소요시간 8분."

### Investigate

행동 전에 알아야 할 핵심 개념을 짧게 설명한다. 공식 자료 기반 출처를 함께 제공한다.

포함 요소:

- 왜 필요한가
- 어떤 사람에게 해당되는가
- 비용과 시간은 어느 정도인가
- 주의할 점은 무엇인가

### Act

사용자가 실제 행동을 수행할 수 있게 단계별 안내와 외부 링크를 제공한다.

예:

- 청약통장 가입 페이지로 이동
- 자동이체 설정 화면 안내
- 연말정산 간소화 서비스 확인
- ISA 계좌 개설 전 체크리스트 확인

### Reflect

완료 후 사용자가 무엇을 했고 무엇을 배웠는지 회고한다. AI는 사용자의 답변을 바탕으로 다음 챌린지를 더 잘 추천한다.

## 완료 검증 방식

기획심의 피드백에서 챌린지 완료를 자가 체크와 회고에만 의존하는 점이 지적되었다. MVP에서는 개인정보와 금융정보 수집 부담을 늘리지 않기 위해 증빙 자동 확인을 넣지 않고, 대신 완료 상태를 더 구조화한다.

- 계획 완료: 사용자가 금액, 날짜, 대상 서비스를 정했지만 실제 외부 앱 실행은 아직 하지 않은 상태
- 실행 완료: 사용자가 외부 앱 또는 실제 생활에서 행동을 수행했다고 체크한 상태
- 보류: 실행하지 못한 이유를 선택하고 다음 시도를 예약한 상태

각 챌린지는 완료 체크 질문, 보류 사유, 다음 행동 후보를 반드시 가진다.

### 실행 완료 게이트 (2026-07-12 구현)

실행 완료는 버튼 한 번이 아니라 3단 자가검증 게이트를 거친다. 증빙을 요구하지 않으면서 "실제로 했는가"를 구조적으로 확인하는 장치다.

1. 스텝 체크리스트 전체 체크 — 체크 전에는 실행 완료 버튼이 비활성화된다.
2. 완료 확인 질문 전원 긍정 — 챌린지별 `completionCheckQuestions` (shared/challenges/mvp-core-challenges.yaml 계약).
3. 회고 1문 입력 — `reflectionQuestions` 첫 문항. 비어 있으면 확정할 수 없다.

유형별 게이트는 다음과 같이 적용한다.

| 유형 | 게이트 |
| --- | --- |
| A 즉시완료 | 동일 시트 (확인 질문 1개 수준으로 가벼움) |
| B 실행 | 동일 시트 (자가체크 + 회고 게이트) |
| C 습관 (하루형) | 스텝 체크리스트가 체크인 기록을 겸함 + 동일 시트 |
| C 습관 (주간 누적형) | 날짜별 체크인 누적 — 해당 챌린지 도입 시 구현 |

질문에 다 긍정하지 못하는 사용자는 빈손으로 내보내지 않고 "아직이에요" 버튼으로 계획 완료(가중치 50%)로 전환한다.

### 증빙(인증샷) 첨부 — 선택 보너스

완료 확인 시트에서 인증샷을 선택적으로 첨부할 수 있다. 원칙:

- 첨부는 항상 선택사항이다. 첨부 없이도 실행 완료 인정에 아무 불이익이 없다.
- 사진은 기기에서만 잠깐 쓰이고 저장·전송하지 않는다. 기록에는 첨부 여부만 남긴다.
- 증빙은 검증되지 않으므로 지킨 돈/예약된 돈 금액에 가산하지 않는다 (진정성 원칙).
- 보상은 배지 「기록으로 남긴 실행」(첫 1회)과 완료 리액션의 보너스 표시로 한정한다.

## 챌린지 분류

- 비상금: 월 고정지출 계산, 비상금 목표 설정, 자동이체 설정
- 청약·연금: 청약통장 확인/개설, 연금저축 이해, IRP 기초 확인
- 적금·예금: 청년 정책금융 탐색, 예적금 금리 비교, 만기 전략 세우기
- 투자 기초: ETF 개념 이해, 모의투자, 투자 성향 점검, 장기투자 원칙 학습
- 자산 분산: 자산 비중 점검, 리밸런싱 이해, 월간 자산 리포트
- 세금/정책: 연말정산 점검, 근로장려금 확인, 청년도약계좌 등 정책금융 확인

## MVP 챌린지

초기 챌린지는 완료율을 가장 중요하게 본다. 사용자가 5~15분 안에 끝낼 수 있고, 결과가 숫자나 체크 상태로 보이는 행동부터 제공한다.

### 1. 비상금 목표 만들기

- 입력: 월 필수 생활비, 목표 개월 수, 현재 비상금
- 완료 기준: 목표 비상금과 월 저축 계획 확정
- 결과: 목표 금액, 현재 달성률, 월 저축 금액별 예상 달성 기간

### 2. 월급날 자동이체 계획 만들기

- 입력: 월급일, 자동이체 금액, 자동이체 날짜
- 완료 기준: 자동이체 계획 저장 또는 은행 앱에서 설정 완료 체크
- 결과: 월 예상 저축액, 연간 누적 원금, 목표 달성 예상 시점

### 3. 고정지출 한 개 줄이기

- 입력: 구독/고정지출 목록, 최근 사용 여부, 해지 후보
- 완료 기준: 하나의 고정지출 해지 또는 해지 예정 등록
- 결과: 월 절감액, 연간 절감액, 누적 절감액

### 4. 청약통장 상태 점검하기

- 입력: 가입 여부, 월 납입 여부, 납입 금액, 납입 날짜
- 완료 기준: 상태 점검 완료와 보완 행동 선택
- 결과: 점검 완료 상태, 보완해야 할 행동, 다음 청약 챌린지

### 5. 현실적인 월 저축률 정하기

- 입력: 월 실수령액, 필수 고정지출, 가용 금액
- 완료 기준: 이번 달 저축률과 저축 목표 확정
- 결과: 월 저축 목표, 연간 누적 원금, 실천 난이도

### 6. 청년 자산 형성 정책 자격 간이 확인하기

- 입력: 나이대, 근로/소득 상태, 거주 지역, 관심 정책
- 완료 기준: 확인할 정책 1개와 다음 확인 날짜 선택
- 결과: 확인 대상 정책, 공식 페이지 링크, 준비할 정보 목록

## 챌린지 데이터 구조

MVP에서는 앱과 API 구현 편의성을 위해 camelCase 필드를 기본으로 사용한다.

- id
- title
- shortDescription
- category
- assetStage
- difficulty
- estimatedMinutes
- requiredItems
- expectedOutcome
- whyContent
- howContent
- cautionContent
- sourceList
- steps
- completionMethod
- impactCalculation
- completionCheckQuestions
- holdReasonOptions
- reflectionQuestions
- nextChallengeCandidates

## 챌린지 라이브러리 파일

- 전체 후보 카탈로그: [shared/challenges/challenge-library-v1.yaml](../shared/challenges/challenge-library-v1.yaml)
- 초기 상세 MVP 시나리오: [shared/challenges/mvp-challenges.yaml](../shared/challenges/mvp-challenges.yaml)
- MVP 코어 16 상세 시나리오: [shared/challenges/mvp-core-challenges.yaml](../shared/challenges/mvp-core-challenges.yaml)
- 기획/운영 설명: [docs/14-challenge-library.md](./14-challenge-library.md), [docs/15-challenge-scenarios.md](./15-challenge-scenarios.md)

`challenge-library-v1.yaml`은 후보 풀이고, 실제 앱에 넣을 챌린지는 `mvp-challenges.yaml` 수준으로 단계, 완료 질문, 회고 질문, 확보효과 산식까지 상세화한다.

## 예시

```yaml
id: housing-subscription-check
title: 청약통장 가입 가능 여부 확인하기
assetStage: 청약·연금
estimatedMinutes: 8
difficulty: easy
whyContent: 청약통장은 주거 준비의 기본 토대이며, 가입 기간이 길수록 유리한 경우가 많다.
steps:
  - 현재 청약통장 보유 여부를 확인한다.
  - 미가입 상태라면 은행 앱에서 주택청약종합저축 메뉴를 찾는다.
  - 월 납입 가능 금액을 정하고 가입 전 유의사항을 확인한다.
completionMethod: 가입 완료 또는 가입 예정 은행과 납입액 결정
reflectionQuestions:
  - 오늘 새롭게 알게 된 점은 무엇인가?
  - 가입을 막은 가장 큰 장벽은 무엇인가?
```
