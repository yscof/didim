# 12. Review Feedback Actions

## 분석 대상

- 문서: 기획심의 종합의견서
- 등록일: 2026-07-02
- 반영 기준: 챌린지 완료율, 재방문율, 사용 편의성에 직접 기여하는 피드백만 MVP에 반영

## 핵심 피드백 요약

- 기존 핀테크 서비스 대비 서비스 차별점이 더 구체적으로 보여야 한다.
- CBL 챌린지가 실제 액션으로 이어지게 만드는 동기부여 설계가 부족하다.
- 사용자가 지속적으로 미션을 수행하게 하는 보상과 재방문 요소가 부족하다.
- 사용자 데이터가 추천 서비스로 연결되는 과정의 신뢰성 검증이 필요하다.
- 챌린지 완료가 자가 체크와 회고에 의존하므로 실제 행동 여부를 더 잘 기록할 방법이 필요하다.
- 개인정보와 금융정보 수집, 저장, 보안 계획이 더 구체적이어야 한다.
- 정부/지자체 청년 자산 형성 정책을 챌린지로 연결하는 유입 전략은 적절하다.
- ETF 검색, 실시간 자산 업데이트, 금융권 API 제휴 등 확장 아이디어는 있으나 MVP 범위와 리스크를 고려해야 한다.

## MVP에 반영한 항목

### 1. 완료 상태 3분류

자가 체크만으로는 실제 행동 완료를 판단하기 어렵다는 피드백을 반영해 완료 상태를 구조화한다.

- 계획 완료
- 실행 완료
- 보류

보류 상태는 실패로 처리하지 않고 보류 사유와 다음 시도 예약으로 이어지게 한다.

### 2. 추천 근거 표시와 로그

사용자의 자산, 소득, 소비, 목표가 어떻게 추천으로 이어지는지 검증 가능해야 한다는 피드백을 반영한다.

- 사용된 입력 상태
- 매칭된 룰
- 제외된 챌린지 후보
- 이번 주 추천 이유

MVP에서는 복잡한 추천 모델보다 설명 가능한 룰 기반 추천을 우선한다.

### 3. AI 개인화 칭찬과 다음 행동 1개

사용자가 챌린지를 지속하게 하는 동기부여가 부족하다는 피드백을 반영한다.

- 사용자가 완료한 행동을 구체적으로 인정한다.
- 다음 행동은 하나만 제안한다.
- 실패나 보류도 다시 시도 가능한 상태로 다룬다.

### 4. 개인정보 최소 수집

실시간 계좌 연동이나 민감 정보 저장은 MVP에서 제외하고, 상태값과 범주형 입력을 우선 사용한다.

예:

- 비상금 부족
- 청약 미점검
- 저축률 미설정
- 고정지출 절감 후보 있음

### 5. 청년 정책 체크 챌린지

청년도약계좌, 근로장려금 등 청년 자산 형성 정책을 챌린지로 연결하는 유입 전략이 적절하다는 피드백을 반영한다. MVP에는 복잡한 신청 자동화가 아니라 자격 간이 확인과 공식 페이지 연결만 포함한다.

## MVP에서 제외한 항목

- ETF 구성 종목/수수료 검색
- 실시간 계좌 연동
- 증빙 자동 확인
- 금융상품 가입 대행
- 금융권 API 제휴 기반 CPA 플로우
- 타깃 확장: 30대 결혼 예정자, 40대 가정 등

위 항목은 가치가 없어서 제외한 것이 아니라, MVP 핵심 지표인 첫 행동 완료율과 2주차 재방문율을 검증하기 전에는 구현 비용과 규제/개인정보 리스크가 크기 때문이다.

## 중간점검 전 확인할 지표

- 첫 챌린지 수락률
- 실행 완료율
- 계획 완료율
- 보류 후 재시도율
- 2주차 재방문율
- 추천 근거 이해도
- 개인정보 입력 부담 점수
- AI 칭찬/회고 만족도
