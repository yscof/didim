# 17. 디딤 개발 현황 (5분 소개)

> 서비스 소개 페이지(`/about`)의 원본 원고. 발표·소개용으로도 사용한다.
> 기준일: 2026-07-13. 배포: https://yscof.github.io/didim/

## 디딤은 무엇인가

디딤은 사회초년생 1~3년차를 위한 금융 행동 코치입니다. 금융 정보는 넘치지만 행동으로 이어지지 않는다는 문제에서 출발해, **매주 하나의 실제 금융 행동을 챌린지로 제안하고 실행까지 안내**합니다. 현재 웹 버전이 배포되어 있고(Flutter 웹, GitHub Pages 자동 배포), 데이터는 브라우저 로컬 저장 기반으로 동작하며 백엔드·AI 코치 연동은 다음 단계입니다.

## 기능

### 1. 챌린지 — 서비스의 심장

매주 하나씩 추천되는 실행형 금융 미션입니다. "공부하세요"가 아니라 "이걸 하세요"입니다.

- **카테고리 탐색**: 소비 절약 / 저축·비상금 / 정책 혜택 / 연말정산. 무신사식 상단 메뉴바와 카테고리 탭으로 둘러볼 수 있습니다.
- **진행 흐름**: 코치의 훅(왜 지금 이게 중요한지 한 문장) → 왜 필요한가 → 단계별 체크리스트 → 완료 확인. 예: "13월의 월급 예약하기"는 문항 6개 응답만으로 내 연말정산 혜택 지도를 만들어주고, 이번 주에 할 행동 딱 하나를 고르게 합니다.
- **완료 3분류**: 실행 완료 / 계획 완료 / 보류. 보류도 실패가 아니라 "사유 기록 + 다음 시도 예약"으로 관리됩니다.
- **금융 여정 지도**: 챌린지를 완료하면 생활비 마을, 비상금 숲, 정책혜택 항구, 연말정산 지역이 완성되어 가는 지도가 자랍니다 (20/40/60/80/100% 마일스톤 장면).

### 2. 가계부

계좌 연동 없는 가벼운 자가 입력 가계부입니다. 이번 달 지출/수입/남은 돈 요약, 카테고리별 지출 막대, 날짜별 내역을 제공합니다. 기록을 강제하는 가계부가 아니라 챌린지(무지출 데이 등)의 실측 기록을 뒷받침하는 보조 도구로 설계했습니다.

### 3. 디지털 월세 (구독료 관리)

매달 나가는 구독료를 "또 하나의 월세"로 보여줍니다. 구독을 등록하면 월 총액과 연 환산 금액이 나오고, 줄일 수 있는 혜택 6가지(연간 결제 전환, 가족·공유 요금제, 통신사 결합 점검, 학생 할인, 멤버십 중복 정리, 해지 전 혜택 확인)를 안내합니다. 가장 확실한 절감법인 "안 쓰는 구독 해지"는 구독 전수조사 챌린지로 바로 연결됩니다.

### 4. 카드 재테크

카드별로 월 실적 기준 대비 이번 달 사용액을 게이지로 보여줍니다 — "얼마 썼고, 앞으로 얼마 더 쓰면 혜택 조건을 채우는지, 월말까지 며칠 남았는지". 발급일과 의무 유지 기간을 입력하면 **해지 가능일 D-day**와 다음 연회비 예상일도 관리됩니다. 카드 소득공제 25% 라인 챌린지와 연결되어 연말정산까지 이어집니다.

도구 3종(가계부·디지털 월세·카드 재테크)은 모두 자가 입력 + 로컬 저장(새로고침에도 유지)이며, 챌린지로 유입되는 입구 역할을 합니다.

## 완료 검증 — 어떻게 "진짜 했는지" 확인하나

**단순 결과 입력도, OCR도 아닙니다. 자가검증 게이트 방식입니다.**

실행 완료를 누르려면 네 단계를 통과해야 합니다:

1. **스텝 전체 체크** — 단계별 체크리스트를 모두 체크해야 완료 버튼이 활성화됩니다.
2. **자가체크 질문** — 챌린지별 완료 확인 질문에 답합니다 ("자동이체를 설정했나요?").
3. **회고 필수** — 한 줄이라도 회고를 써야 확정됩니다. 회고는 다음 추천의 재료가 됩니다.
4. **실측 입력** — 금액형 챌린지는 실제 수치(해지한 구독료, 자동이체 금액 등)를 입력해야 하고, 확보효과는 그 자리에서 `입력값 × 산식`으로 계산됩니다. 고정 예시 금액은 쓰지 않습니다.

인증샷 첨부는 **선택**이며 보너스 배지에만 영향을 주고, 이미지는 저장하지 않고 첨부 여부만 기록합니다(개인정보 최소 수집 원칙).

왜 이 방식인가: MVP에서 계좌 연동·증빙 자동 확인은 개인정보 부담과 규제 비용이 커서 제외했습니다. 대신 **거짓의 비용을 높이고 이득을 없애는 설계**를 택했습니다 — 거짓 수치를 입력하면 본인의 자산 기록(지킨 돈 카운터, 지도)이 오염될 뿐 얻는 것이 없습니다. 검증 강도는 사다리로 준비되어 있습니다: 자가검증 게이트(현재) → 후속 확인 → AI 인증샷 판독 → 계좌 연동. 현금성 보상을 도입하는 날이 온다면 상위 단계 검증이 선행 조건입니다.

## 보상 시스템 — 완료하면 무엇을 받나

**현금·포인트·실물 보상은 없습니다. 의도된 설계입니다.**

디딤의 점수판은 가짜 화폐가 아니라 **사용자 본인의 진짜 돈**입니다:

- **이중 카운터**: "지킨 돈"(무지출·해지 등 실제 발생한 절약만 합산)과 "예약된 돈"(연말정산 환급 예상액 등 추정·연 환산 별도 트랙). 클릭하면 어떤 챌린지로 얼마가 쌓였는지 상세 내역이 나옵니다. 추정액을 실적으로 과장하지 않는 것이 원칙입니다.
- **금융 여정 지도**: 완료할 때마다 지역에 오브젝트가 생기고 색이 채워집니다. 지도 완성 자체가 보상이 되는지가 핵심 검증 가설입니다.
- **배지**: 행동 배지(첫 금융 행동, 자동저축 시작 등) / 지역 완성 배지 / 꾸준함 배지. 모든 챌린지에 주지 않고 의미 있는 행동에만 지급하며, 도감이 아니라 "획득 날짜 + 실제 행동"이 남는 성장 일지 형태입니다.
- **주간 스트릭**: 주 1개 이상 완료하면 이어집니다. 성실한 보류(사유 기록+재시도 예약)도 유지로 인정하고, 끊겨도 기록을 초기화하지 않습니다. 압박이 아니라 복귀를 돕는 설계입니다.
- **완료 리액션**: 완료 직후 5~8초 — 축하 → 실제 금융 효과 → 지도 변화 → (조건 시) 배지 → 다음 챌린지 1개 추천.

**"같은 나이 상위 몇 %" 같은 또래 비교·랭킹도 넣지 않았습니다.** 돈의 비교는 동기부여보다 비참함을 만들기 쉽고, 벤치마크 분석에서도 현금성·경쟁형 보상은 실패 패턴(보상 경제 붕괴, 규제 리스크)이었습니다. 디딤의 비교 대상은 남이 아니라 **과거의 나**입니다 — 순자산 추적, 1년 뒤 나에게 쓰는 편지, 월간 여정 리포트가 그 장치입니다. 예외적으로 검토 중인 확장은 두 가지뿐입니다: 개인정보가 드러나지 않는 완료 카드 공유, 그리고 (베타 이후) Ant Forest식 "내 성과가 검증 가능한 실물 기여로 전환"되는 이타적 보상.
