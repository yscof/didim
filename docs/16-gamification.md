# 16. Gamification — 챌린지 완료 보상 시스템

## 목적

챌린지를 완료할수록 **금융 여정 지도가 실제로 완성되는 구조**로 완료 보상과 재방문 이유를 만든다. 단순 포인트가 아니라 사용자가 자신의 금융 기반이 만들어지는 모습을 계속 확인하게 하는 것이 목표다.

기존 결정([08-decisions.md](./08-decisions.md))과의 정합:

- 무작위 보상·포인트 상점·친구 초대 리워드는 쓰지 않는다.
- 점수판의 본체는 가짜 화폐가 아니라 **진짜 돈**(확보효과 카운터)이다. 지도와 배지는 그 위의 상징 레이어다.
- 비교 대상은 남이 아니라 과거의 나다. 랭킹·리더보드는 만들지 않는다.

구조화 데이터:

- 지도 정의: [shared/gamification/journey-map.yaml](../shared/gamification/journey-map.yaml)
- 배지 정의: [shared/gamification/badges.yaml](../shared/gamification/badges.yaml)

## 핵심 루프

```text
이번 주 챌린지 추천
→ 실제 금융 행동
→ 완료 리액션 (5~8초)
→ 지도 변화
→ 배지 획득 (조건 달성 시)
→ 다음 지역·챌린지 해금
```

지도는 지속적인 성장과 탐색을, 배지는 중요한 순간의 기록을, 완료 리액션은 즉각적인 성취감을 담당한다. 세 요소는 하나의 흐름으로 연결된다.

## 1. 금융 여정 지도

사용자의 금융생활을 하나의 여행 지도로 표현한다. 처음에는 대부분의 지역이 흐리거나 잠겨 있고, 챌린지를 완료하면 길이 생기고 건물이 세워지고 지역에 색이 채워진다.

```text
[출발지] 금융상태 점검
    ├─ 생활비 마을        (소비 습관 · 고정비)
    ├─ 비상금 숲          (비상금)
    └─ 정책혜택 항구      (세금 · 정책 · 청약)
         ↓ (베타)
    신용·부채 요새 / 저축 평원
         ↓ (후반)
    투자 전망대
```

### 지역 구성과 챌린지 매핑

MVP 코어 16개([mvp-core-challenges.yaml](../shared/challenges/mvp-core-challenges.yaml))를 지역에 매핑한다. 상세 매핑·오브젝트·진행 가중치는 journey-map.yaml에 있다.

| 지역 | 챌린지 | 상태 |
|---|---|---|
| 출발지 (금융상태 점검) | O-1 소비 캐릭터, O-2 통장 이름, M-4 순자산, M-16 편지(타임캡슐) | MVP |
| 생활비 마을 | M-1 생존 테스트, M-5 구독, M-10 통신비 (메인) / M-6 배달, M-7 무지출, M-8 보류, M-9 시급 (사이드) | MVP |
| 비상금 숲 | M-2 목표(씨앗), M-3 통장 분리+자동이체(첫 나무), 한 달 유지(쉼터) | MVP |
| 정책혜택 항구 | M-12 놓치는 돈(배 입항), M-15 청약(등대·새 항로) | MVP |
| 연말정산 | M-13 13월의 월급(환급 지도) + 후속 4개: 중기 감면 신청, 월세 서류, 연금저축, 카드 25% 라인 | MVP |
| 신용·부채 요새 | M-14 신용점수 + 확장(대출 전략, 보험 점검) | 베타 (실루엣 예고) |
| 저축 평원 | M-11 예적금 비교 + 확장(단리복리 등) | 베타 (실루엣 예고) |
| 투자 전망대 | 4단계 확장 챌린지 | 후반 (기반 완성자 해금) |

베타 지역 소속 챌린지(M-11, M-14)는 MVP에서도 추천 리스트로 수행할 수 있고, 완료 기록은 지역이 열릴 때 진행도로 소급 반영된다.

### 분기형 경로

모든 사용자가 같은 순서를 밟지 않는다. 온보딩 진단 결과에 따라 추천 경로가 달라진다.

- 비상금 없는 사용자: 생활비 점검 → 비상금 숲 우선 (M-1 → M-2 → M-3 → 한 달 유지)
- 정책 관심 사용자: 정책혜택 항구 즉시 해금 (M-12 → 후속 미션 → M-13/M-15)

지도에서 다음 지역 해금 조건은 "완성"이 아니라 **일부 진행**(예: 40%)으로 잡아, 한 지역이 지루해도 갈아탈 수 있게 한다.

### 메인 / 사이드 챌린지

- **메인**: 금융 상태를 실제로 개선하는 핵심 행동(B형 위주). 지도에 큰 변화(대표 오브젝트)를 만든다.
- **사이드**: 짧고 가벼운 보조 행동(A·C형, 스캔형의 후속 미션). 작은 장식 오브젝트와 진행도를 제공한다.

## 2. 진행도 규칙

완료 상태 3분류([02-challenge-system.md](./02-challenge-system.md))와 정합시킨다.

- **실행 완료(executed)** = 챌린지 가중치 100% 반영
- **계획 완료(planned)** = 가중치 50% 반영, 실행 체크 시 나머지 반영 → "계획만 세워도 지도가 조금 자란다"
- **보류(held)** = 진행도 없음. 대신 지도에 **예약 표지판**이 꽂힌다("다음 시도: 7/16"). 보류가 빈손이 아니라 지도 위의 약속이 된다.

지역 중간 변화(완성까지 기다리게 하지 않는다):

| 진행도 | 변화 |
|---|---|
| 20% | 길 생성 |
| 40% | 작은 건물 생성 + 다음 지역 해금 |
| 60% | 대표 오브젝트 생성 |
| 80% | 지역 활성화 (불빛·주민) |
| 100% | 지역 완성 + 지역 배지 |

## 3. 완료 리액션 (5~8초)

챌린지 완료 직후 5단계를 한 흐름으로 보여준다. 어느 단계에서든 스킵 가능해야 한다.

1. **완료 인정** — 체크 애니메이션 + 한 줄 축하. "자동이체 설정 완료! 비상금을 만드는 가장 중요한 첫 행동을 끝냈어요."
2. **실제 금융 효과** — gain 라벨 규칙([15-challenge-scenarios.md](./15-challenge-scenarios.md))대로 표시. "매달 20만 원씩 모으면 1년 후 240만 원 (연 환산)". 금액을 특정할 수 없으면 `예상`, `조건에 따라 달라질 수 있음`을 명시한다. `최대` 단독 강조는 진정성 원칙에 따라 쓰지 않는다.
3. **지도 변화** — 카메라가 지도로 이동해 해당 지역이 변한다. "비상금 숲 25% → 45%. 새로운 길이 열렸습니다."
4. **배지 획득** — 조건 달성 시에만 별도 연출. 모든 챌린지에 배지를 주지 않아야 가치가 유지된다.
5. **다음 챌린지 연결** — 다음 행동을 **한 가지만** 추천. 버튼은 [다음 챌린지 확인] [지도 구경하기] 두 개. 축하 화면에서 막히지 않고 다음 행동으로 흐르게 한다.

2단계 금융 효과는 이중 카운터에 누적된다: **지킨 돈**(realized 실발생만 합산)과 **예약된 돈**(estimated·annualized 별도 트랙).

## 4. 배지 시스템

세 종류, MVP 10개. 정의는 [badges.yaml](../shared/gamification/badges.yaml).

- **행동 배지 (5)** — 핵심 행동 최초 완료: 첫 금융 행동 / 자동저축 시작 / 혜택 탐험가 / 고정비 다이어트 / 신용 확인 완료
- **지역 배지 (3)** — 지역 완성: 생활비 설계자 / 안전망 건설자 / 혜택 항해사
- **꾸준함 배지 (2)** — 주간 루틴 유지: 2주 금융 루틴 / 4주 금융 루틴. 로그인·출석이 아니라 **챌린지 완료 주** 기준이다.

### 배지 화면은 도감이 아니라 성장 기록

배지 나열이 아니라 획득 날짜와 실제 행동을 함께 기록한다.

> **자동저축 시작** · 2026년 7월 15일
> 매월 비상금 자동이체를 설정했어요.

이 화면이 곧 사용자의 **금융 성장 일지**가 된다. 미획득 배지는 숨기지 않고 조건 일부를 보여준다("비상금 챌린지 3개를 완료하면 획득"). 단, 공유 카드에서는 금액·자산 규모가 드러나는 정보를 자동 제외한다(`shareSafe` 필드).

## 5. 홈 화면 구성

정보 나열이 아니라 지금 할 행동에 집중시킨다. 가장 중요한 버튼은 항상 하나다.

```text
OO님의 금융 여정

[이번 주 챌린지 시작하기]  ← 단일 핵심 버튼
비상금 통장 분리 + 자동이체 · 약 20분 · 비상금 숲 +45%

금융 여정 지도 (미니맵)
생활비 마을 80% · 비상금 숲 35% · 정책혜택 항구 20%

지킨 돈 128,700원 · 예약된 돈 91만 원 (예상)
최근 배지: 혜택 탐험가

[전체 지도 보기]
```

## 6. 재방문 장치

지도와 배지만으로 리텐션이 생기지 않는다. 함께 적용할 구조:

- **주 1회 핵심 챌린지** — 기존 결정(주 1개 기본값) 유지. "이번 주에는 비상금 숲에 첫 나무를 심어보세요."
- **다음 지역 예고** — 잠긴 지역 일부를 보여준다. "비상금 숲을 60% 완성하면 정책혜택 항구의 새 경로가 열려요."
- **중간 변화** — 위 20/40/60/80/100 마일스톤. 챌린지 하나마다 눈에 보이는 변화가 있어야 한다.
- **월간 금융 여정 리포트** — "7월: 챌린지 4개 완료 · 신규 지역 1개 해금 · 연간 예상 절약액 118,800원 · 새 배지 2개". 지난 방문 이후 변화 카드([13-growth-retention.md](./13-growth-retention.md))의 월간 확장판.
- **시즌 트리거** — M-13의 10월 말·1월 재방문 트리거는 항구 지역 이벤트("등대 점검의 날")로 표현할 수 있다.

### 주간 스트릭 (보조 요소)

- 기준: 매주 **하나 이상의 핵심 챌린지 완료** 시 이어진다. 일일 접속 기준 금지.
- **보류도 유지로 인정**: 보류 사유 기록 + 재시도 예약까지 하면 그 주의 루틴을 지킨 것으로 본다 (성실한 보류 = 리듬 유지).
- 끊겨도 초기화하지 않는다: "지난주는 잠시 쉬었어요. 이번 주 챌린지를 완료하면 여정을 다시 이어갈 수 있어요." 지나온 지도와 배지는 그대로 남는다.
- 압박이 아니라 복귀를 돕는 표현만 사용한다 ([01-product-requirements.md](./01-product-requirements.md) 원칙).
- **구현 상태 (2026-07-12)**: 홈(모바일·웹)에 주간 스트릭 카드 구현 — 주별 도트(✅ 완료 / ⏳ 성실한 보류 / 💤 쉼 / 🎯 이번 주), 성실한 보류 유지 인정, 2주 루틴 배지 연동. 과거 주는 목 히스토리 시뮬레이션이며, 성실한 보류의 사유 기록 게이트는 보류 사유 입력 구현 시 강화한다.

## 7. MVP 구현 범위

| 포함 | 내용 |
|---|---|
| 지도 | 출발지 + 3개 지역 (생활비 마을·비상금 숲·정책혜택 항구), 지역별 3~7개 챌린지, 완료 시 오브젝트 변화, 다음 지역 잠금·해금, 베타 지역 실루엣 |
| 배지 | 행동 5 + 지역 3 + 꾸준함 2, 성장 기록형 상세 화면 |
| 완료 리액션 | 5단계 (완료 인정 → 금융 효과 → 지도 변화 → 배지 → 다음 챌린지) |
| 카운터 | 지킨 돈 / 예약된 돈 이중 카운터 |
| 스트릭 | 주간 스트릭, 보류 인정, 비초기화 복구 |

**MVP 제외**: 캐릭터 성장, 포인트 상점, 친구 경쟁·랭킹, 일일 출석 보상. 먼저 **지도 완성 자체를 보상으로 느끼는지** 검증한 뒤 확장한다.

지도 오브젝트 연출 수준(정적 일러스트 vs 애니메이션)은 9월 일정에 맞춰 별도 결정한다 ([09-open-questions.md](./09-open-questions.md)).

**지도 구현 상태 (2026-07-12)**: 그 전 단계의 플레이스홀더로, 지역 카드에 마일스톤(20/40/60/80/100%) 이모지 장면을 구현했다. 미도달 마일스톤은 흐린 예고로 보여주고, 완료 리액션에서 새로 도달한 오브젝트에 1회 등장 연출을 넣었다. 지역 잠금(unlockCondition)은 온보딩 진단 미구현으로 아직 적용하지 않았다(전 지역 노출, 소급 반영 규칙 덕에 후행 도입 가능).

|---|---|---|---|---|---|
| 생활비 마을 | 🛤️ | 🏠 | 🏪 | 🏮 | 🏘️ |
| 비상금 숲 | 🌱 | 🌿 | 🌳 | 🦌 | 🏕️ |
| 정책혜택 항구 | 🌊 | ⛵ | ⚓ | 🗼 | 🚢 |

## 7.5 벤치마크 검증 (2026-07-12)

보상 시스템 확장 검토에서 성공·실패 사례를 분석한 결과, 이 문서의 설계 원칙이 성공 패턴과 일치함을 확인했다 (상세 근거: [08-decisions.md](./08-decisions.md) 2026-07-12 보상 결정).

| 성공 패턴 | 사례 | 디딤 대응 |
|---|---|---|
| 최소 행동의 즉시 가시화 | Fortune City (기록 1건=건물 1채) | 채택 — 지도 마일스톤 장면 |
| 손실회피 + 안전망 | Duolingo (스트릭+프리즈) | 채택 — 주간 스트릭 + 성실한 보류 인정·비초기화 |
| 누적물의 정체성화 | Strava Year in Sport | 설계 있음 — 변화 카드·월간 리포트 (후속 구현) |
| 실물·사회적 전환의 신뢰 | Ant Forest (실제 나무 6.19억 그루) | 베타 검토 — 이타적 실물 전환, 신뢰 가능한 파트너 필요 |
| 가벼운 상호 인정 | Strava kudos | 다음 후보 — shareSafe 완료 카드 공유 |
