# 18. Compliance — 금융 규제 리스크 맵과 대응

> ⚠️ 이 문서는 법률 자문이 아니다. 팀 내부의 설계 기준이며, 정식 출시 전
> 법률·컴플라이언스 전문가 검토를 받아야 한다. 검토 시점: 베타 공개 전 필수.

## 핵심 포지션

**디딤은 금융상품의 판매·중개·자문이 아니라 일반적 금융 정보·교육을 제공하는 서비스다.**
모든 기능·콘텐츠·문구는 이 포지션을 벗어나지 않아야 하며, 벗어나는 기능(계좌 연동,
현금 보상, 제휴 등)을 도입할 때는 아래 트리거 표의 조치가 선행되어야 한다.

사용자에게 보이는 고지 체계: Footer 상시 면책 + 면책 고지(/disclaimer) ·
이용약관(/terms) · 개인정보처리방침(/privacy) 페이지.

## 규제 리스크 맵

### 1. 자본시장법 — 투자자문업·유사투자자문업

- **걸리는 지점**: 특정 종목·시점의 매매 판단을 제공하거나, 대가를 받고 투자 조언을
  반복 제공하면 등록 대상. "이 ETF를 사세요" 류의 표현이 위험.
- **현재 조치**: 챌린지 라이브러리의 격리 원칙 — 실매수 유도, 특정 종목 판단,
  유망 산업 선택 유도 챌린지는 서비스에서 제외 상태 유지
  ([challenge-library-v1.yaml](../shared/challenges/challenge-library-v1.yaml)의
  isolatedOrExcluded). 투자 콘텐츠는 교육·계산·시뮬레이션·관찰형만 제공하고
  "판단과 책임은 본인" 고지를 붙인다.
- **금지 문구 예**: "지금 사야 할", "유망한", "추천 종목". → "비교해보기",
  "계산해보기", "관찰하기"로만 표현.

### 2. 금융소비자보호법 — 판매·중개·광고

- **걸리는 지점**: 금융상품 판매대리·중개는 등록 없이 불가. 특정 상품으로의 가입
  유도·연결이 중개로 해석될 수 있고, 금융상품 광고에는 심의·표기 의무가 있다.
- **현재 조치**: ① 특정 금융상품명 비노출 — 카테고리 수준까지만 안내 (예: "파킹형
  통장", "청년 대상 적금") ② 상품 비교는 금융감독원 금융상품통합비교공시 등
  **공공 비교공시 링크로만** 연결 ③ 가입 대행·신청 자동화 없음 (MVP 제외 확정)
  ④ 확보효과 금액에 '추정' 라벨, '최대' 단독 강조 금지.
- **향후 트리거**: CPA(가입 성과형) 제휴 도입 시 — "광고" 표시 의무, 금융상품
  광고 심의 여부 검토, 이해상충 고지가 선행 조건 (docs/09 오픈 퀘스천).

### 3. 신용정보법·마이데이터 (본인신용정보관리업)

- **걸리는 지점**: 계좌·카드 내역 등 개인신용정보를 수집·통합 조회하는 서비스는
  마이데이터 허가 대상. 스크래핑은 금지.
- **현재 조치**: 전 기능이 **자가 입력 + 브라우저 로컬 저장**으로 개인신용정보를
  수집하지 않는다 (가계부·구독·카드 실적 모두). 이 구조 자체가 규제 회피가 아니라
  적법한 비해당 상태다.
- **향후 트리거**: 계좌 연동 도입 시 — 자체 허가가 아니라 **허가받은 마이데이터
  사업자 API 제휴** 방식으로만 검토. 서버 저장 도입 시 신용정보 해당 여부 재검토.

### 4. 개인정보보호법

- **걸리는 지점**: 금융 관련 정보는 민감도가 높고, 수집·이용에 동의와 안전조치가
  필요하다.
- **현재 조치**: ① 회원가입·서버 전송 없음, 입력 데이터는 localStorage에만 저장
  ② 챌린지 인증샷은 이미지 미저장(첨부 여부만 기록) ③ 개인정보처리방침 게시
  (/privacy) ④ 분석 도구·광고 쿠키 미사용.
- **향후 트리거**: 계정·서버 저장 도입 시 — 수집 항목 최소화 재설계, 동의 절차,
  암호화, 처리방침 개정·고지. 분석 도구(GA 등) 도입 시 방침 개정.

### 5. 세무사법 등 자격사 업무 (세무·법률 상담)

- **걸리는 지점**: 개별 사안에 대한 세무 상담·대리는 세무사 업무. 연말정산 챌린지가
  "당신은 얼마 환급받습니다"를 단정하면 위험.
- **현재 조치**: 일반적 제도 안내 + 응답 기반 '추정' 표기 + "회사·홈택스·전문가
  최종 확인" 고지 (M-13 스펙의 원칙을 전 세금 콘텐츠로 일반화). 개별 상담 기능 없음.

### 6. 표시·광고의 공정화 (과장 금지)

- **걸리는 지점**: 효과 과장, 수익 보장 표현.
- **현재 조치**: '최대 ○○원' 남용 금지, 추정·연 환산 라벨과 연도 표기, "확정",
  "보장" 표현 금지, 실발생/추정 트랙 분리 (지킨 돈/예약된 돈).

### 7. AI 코치 (향후, docs/03 연계)

- **걸리는 지점**: LLM의 환각·단정 답변이 투자 조언이나 상품 권유로 해석될 위험.
- **조치 설계**: 공식 자료 기반 RAG + 출처 표시, 상품 권유·종목 판단성 응답 차단
  가드레일, 면책 문구 상시 노출, 답변-출처 불일치 시 차단 (docs/03).

## 기능 도입 → 선행 조치 트리거 표

| 도입하려는 것 | 선행 조치 |
|---|---|
| 현금성·경품 보상 | 강화 검증(계좌 연동 수준) + 경품·사행성 규제 검토 + 진정성 원칙 재검토 (현재는 미도입 결정) |
| 계좌·카드 연동 | 마이데이터 사업자 제휴, 개인정보 영향평가, 처리방침 개정 |
| CPA·제휴 수익화 | "광고" 표시, 금융상품 광고 심의 검토, 이해상충 고지 |
| 커뮤니티·공유 확장 | 유사투자자문성 게시물(종목 리딩 등) 관리 정책 |
| 서버 저장·계정 | 개인정보 동의 절차, 암호화, 약관·방침 개정 |
| 자산 컨설턴트 중개 | 자격·이해상충·리뷰 신뢰성·책임 검토 (docs/13 후순위 조건 참조) |

## 운영 수칙 (콘텐츠 작성 시)

1. 상품이 아니라 **행동과 제도**를 말한다 ("○○적금 가입" ✗ → "청년 대상 적금
   비교공시에서 확인" ✓)
2. 금액은 항상 추정 라벨 + 기준 연도. 세율·한도는 `validUntil`·`sourceUpdatedAt`으로
   최신성 관리 (shared/challenges의 정책류 필드)
3. 모든 정책·세금 콘텐츠에 공식 기관 링크와 "최종 확인" 문구
4. 새 챌린지 추가 시 이 문서의 리스크 맵을 체크리스트로 사용
