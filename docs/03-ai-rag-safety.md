# 03. AI, RAG, Safety

## AI 역할

AI는 사용자의 금융 이해를 돕고 행동을 유도하는 개인화 코치다. 단순 질의응답 챗봇이 아니라 진단 설명, 미션 개인화, 쉬운 용어 설명, 회고 대화를 담당한다.

## 하이브리드 구조

- 룰 엔진: 5단계 자산 형성 모델, 기본 우선순위, 챌린지 매핑
- RAG: 금융감독원, 국세청, 서민금융진흥원 등 공식 자료 검색
- LLM: 입력 구조화, 쉬운 설명, 개인화 코칭, 회고 대화
- Function Calling: 진척 기록, 챌린지 완료 처리, 다음 챌린지 트리거

## 추천 신뢰성

기획심의 피드백에서 사용자 입력 데이터를 추천으로 연결하는 과정의 신뢰성 검증이 필요하다는 의견이 있었다. MVP에서는 추천 알고리즘을 복잡하게 만들기보다, 추천 근거를 남기고 평가 가능한 구조로 시작한다.

- 추천은 5단계 룰 엔진의 우선순위를 먼저 따른다.
- LLM은 챌린지를 새로 창작하지 않고 검증된 챌린지 라이브러리 안에서 선택 또는 설명을 보조한다.
- 추천 결과에는 사용된 입력 상태, 매칭된 룰, 제외된 후보, 추천 이유를 저장한다.
- 같은 단계의 사용자에게도 보류 사유, 완료 이력, 이해도에 따라 안내 문구와 다음 후보를 다르게 제공한다.
- 추천 품질은 챌린지 수락률, 완료율, 보류 사유, 다음 주 재방문율로 평가한다.

## 파이프라인

1. 입력: 자산, 연봉, 소비, 목표 정보를 계좌 연동 없이 간편 입력
2. 구조화: LLM이 자연어 입력을 필드로 정리
3. 검색: 공식 금융 자료와 단계별 가이드를 RAG로 검색
4. 진단: 룰 엔진이 5단계 모델에서 현재 집중 단계를 결정
5. 설명: LLM이 진단 근거와 다음 행동을 자연어로 설명
6. 챌린지 개인화: 미션 라이브러리에서 이번 주 챌린지 선택
7. 실행 지원: 질문 답변, 출처 표기, 진척 기록, 회고 진행

## 안전 원칙

- 특정 상품의 수익률을 보장하지 않는다.
- 투자 매수/매도 판단을 직접 지시하지 않는다.
- 금융상품 가입 대행이 아니라 사용자가 직접 판단하고 실행하도록 안내한다.
- 정책, 세금, 상품 조건은 기준일과 출처를 표시한다.
- 불확실한 정보는 확인 필요 상태로 답변한다.
- 개인 식별정보와 민감 금융정보는 최소 수집한다.
- 원본 금융정보 저장보다 상태값과 범주형 입력을 우선한다.
- 개인정보 보호법 준수 항목을 개발 체크리스트로 관리한다.

## 답변 스타일

- 초보자가 이해할 수 있는 단어를 우선한다.
- "지금 할 일"을 한 문장으로 정리한다.
- 정보와 행동을 분리해서 보여준다.
- 출처가 없는 금융 사실은 단정하지 않는다.

## 검증 과제

- RAG 소스 목록과 갱신 주기 정의
- 금감원/국세청/서민금융진흥원 자료 수집 방식 확정
- 금융 자문/중개 오해를 줄이는 면책 문구 설계
- AI 응답 평가 세트 구축
- 위험 답변 차단 규칙 작성
- 추천 근거 로그와 추천 품질 평가 세트 구축
- 개인정보 최소 수집 체크리스트 작성
- 트래픽 집중 시점의 비용과 Prompt Caching 적용 가능성 검토
